## 🎯 هدف المرحلة
تجهيز جهاز أبو دجانة بحيث تعمل عليه كل الأدوات المطلوبة (PyTorch نسخة
CPU، مكتبة البيانات، تتبّع التجارب) داخل **بيئة معزولة** لا تفسد بقية
النظام، والتأكد أن كل شيء يعمل بسكربت اختبار.

## 🧠 خلفية مبسّطة
- **البيئة المعزولة (venv — Virtual Environment):** صندوق مستقل
  لتثبيت المكتبات بداخله فقط، حتى لا تتعارض مع مكتبات النظام أو
  مشاريع أخرى. تحذفه بحذف مجلد واحد إن أردت البدء من جديد.
- **PyTorch:** المكتبة الأساسية التي نبني بها الشبكات العصبية.
  لها نسختان: واحدة للكروت (CUDA) وواحدة للمعالج (CPU). نحن CPU.
- **datasets:** مكتبة من HuggingFace لتحميل بيانات نصوص جاهزة
  (مثل WikiText) بسطر واحد.

## ✅ المتطلبات قبل البدء
- لا شيء. هذه أول مرحلة.
- تأكد فقط أن `python3 --version` تطبع 3.12 تقريباً.

## 📋 المهام بالتفصيل

### المهمة 0.1 — إنشاء بيئة معزولة venv باسم wameed-env
**الخطوات:**
1. من داخل مجلد المشروع، نفّذ:
   ```bash
   python3 -m venv wameed-env        # ينشئ البيئة المعزولة
   source wameed-env/bin/activate    # يفعّلها (يظهر اسمها في الترمنال)
   python -m pip install --upgrade pip   # يحدّث أداة التثبيت pip
   ```
2. أنشئ ملف `.gitignore` يحوي سطر `wameed-env/` حتى لا تُرفع البيئة
   على GitHub لاحقاً (حجمها كبير ولا داعي لرفعها).

**معيار القبول (بالأرقام):**
- الأمر `which python` يطبع مساراً يحتوي `wameed-env`.
- لا توجد رسالة خطأ. كود الخروج = 0.

**أخطاء شائعة:**
- `venv: command not found` ⇒ ثبّت الحزمة: `sudo apt install python3-venv`.

---

### المهمة 0.2 — تثبيت PyTorch نسخة CPU فقط
**الخطوات:**
1. مع تفعيل البيئة، نفّذ **بالضبط** (لاحظ رابط CPU):
   ```bash
   pip install torch --index-url https://download.pytorch.org/whl/cpu
   ```
   هذا يضمن نسخة المعالج فقط (أخفّ، بدون CUDA).

**معيار القبول (بالأرقام):**
- هذا الأمر يطبع `False` (أي لا يوجد كارت — وهذا صحيح ومطلوب):
  ```bash
  python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
  ```
- النتيجة المتوقعة: رقم إصدار (مثل 2.x.x) ثم `False`.

**أخطاء شائعة:**
- لو ظهر `True` ⇒ ثُبّتت نسخة CUDA بالخطأ. أزِلها (`pip uninstall torch`)
  وأعد التثبيت بالرابط أعلاه.
- بطء التحميل أو انقطاعه ⇒ أعد المحاولة؛ الحزمة حجمها كبير نسبياً.

---

### المهمة 0.3 — تثبيت datasets + إعداد تتبّع CSV بسيط
**الخطوات:**
1. ثبّت المكتبات الخفيفة:
   ```bash
   pip install datasets pandas psutil
   ```
   - `pandas`: لقراءة/كتابة ملفات CSV بسهولة.
   - `psutil`: لقياس استهلاك الرامة (مهم جداً لجهازنا).
2. أنشئ مجلد `results/` وملف دالة صغيرة `wameed/utils/tracker.py`
   تكتب صفاً جديداً في `results/experiments.csv` بالأعمدة:
   `date, experiment, seed, params, metric_name, metric_value, seconds, peak_ram_mb`.

**معيار القبول (بالأرقام):**
- استدعاء تجريبي للدالة يضيف صفاً، وقراءة الملف تُظهر **صفاً واحداً**
  بالأعمدة الثمانية الصحيحة.

**ملاحظة:** لا نستخدم أدوات تتبّع ثقيلة (مثل Weights & Biases) الآن —
CSV يكفي ولا يستهلك رامة.

---

### المهمة 0.4 — سكربت اختبار البيئة
**الخطوات:**
1. أنشئ `check_env.py` يطبع بالعربية:
   - السطر: «البيئة جاهزة».
   - رقم إصدار PyTorch.
   - عدد أنوية/خيوط المعالج (`os.cpu_count()`).
   - الرامة المتاحة بالميجابايت (عبر `psutil`).
   - `torch.cuda.is_available()` (يجب أن يكون False).

**معيار القبول (بالأرقام):**
- تشغيل `python check_env.py` يطبع كل البنود أعلاه **بدون أي خطأ**،
  وتظهر «البيئة جاهزة»، وعدد الأنوية = 4 (متوقع لجهازه)، و CUDA = False.
