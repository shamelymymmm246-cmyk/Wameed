## 🧠 خلفية مبسّطة + التعريف الدقيق للفكرة

**ما هو SSM ببساطة؟**
نموذج يقرأ النص توكناً توكناً (token — وحدة نصية، كلمة أو جزء كلمة)،
ويحتفظ بـ«حالة» (state — متجه أرقام يلخّص ما قرأه حتى الآن). القاعدة
الأساسية للتحديث (بصورة مبسّطة):

```
h_t = A · h_{t-1} + B · x_t        # الحالة الجديدة = جزء من القديمة + المدخل الجديد
y_t = C · h_t                      # المخرَج من الحالة
```
حيث `h` الحالة، `x` المدخل، و `A,B,C` مصفوفات أوزان تتعلّمها الشبكة.

**ما هو وسم الأهمية (تعريفنا الدقيق)؟**
نضيف رأساً صغيراً (head — طبقة خطية صغيرة) يحسب لكل خطوة **درجة أهمية**:

```
s_t = sigmoid( w · h_t + b )       # رقم بين 0 و1 لكل خطوة زمنية
```
ثم نجعل **بقاء الحالة يعتمد على الأهمية**: الحالات المهمة تبقى أطول،
التافهة تتلاشى أسرع. أبسط صيغة عملية (بوابة احتفاظ):

```
h_t = s_t · ( A · h_{t-1} ) + B · x_t
```
- إذا `s_t ≈ 1`: نحتفظ بالماضي كاملاً (المعلومة مهمة).
- إذا `s_t ≈ 0`: ننسى الماضي عمداً (المعلومة تافهة) ونبدأ من المدخل.

> ملاحظة مفاهيمية: هذه الفكرة قريبة من «البوابات» الموجودة في LSTM
> ومامبا، لكن **ميزتنا**: درجة الأهمية صريحة، قابلة للقراءة والتفسير،
> ويمكن لاحقاً ربطها بالذاكرة الهرمية والضغط (المراحل 2 و3).

**الهدف المساعد (Auxiliary Loss) — خطة احتياطية لو الوسم لم يتعلّم:**
نضيف للخسارة عقوبة تشجّع **التناثر** (sparsity): أن تكون أغلب الدرجات
منخفضة وقليل منها مرتفع، فيُجبَر النموذج على «اختيار» ما يبقيه:
```
الخسارة_الكلية = خسارة_التنبؤ + λ · متوسط(s_t)
```
`λ` معامل صغير (نبدأ بـ 0.01). نفعّله فقط إذا فشل الوسم بدونه.

### المهمة 1.2 — إضافة طبقة الوسم

**لماذا هذا الترتيب؟** نضيف الوسم فوق النموذج الجاهز لنعزل أثره تماماً.

**الخطوات التفصيلية:**

**الخطوة أ — تعديل `wameed/engine/ssm.py`:**

أضف داخل كلاس `TinySSM` المعاملات التالية في `__init__`:
```python
# --- مُعاملات وسم الأهمية ---
self.use_importance = use_importance   # علَم لتفعيل/إيقاف الوسم
if use_importance:
    # طبقة خطية صغيرة: تأخذ الحالة h وتخرج رقماً بين 0 و1
    self.importance_head = nn.Linear(state_dim, 1)
    # الانحياز الأوّلي: نضبطه على -2 حتى يبدأ النموذج بـ s≈0.12
    # هذا يمنع النموذج من «تجميد» الحالة منذ البداية
    nn.init.constant_(self.importance_head.bias, -2.0)
```

**الخطوة ب — تعديل دالة forward:**

بدّل تحديث الحالة من:
```python
h = torch.tanh(self.A(h_prev) + self.B(x_t))
```
إلى:
```python
h_candidate = self.A(h_prev) + self.B(x_t)   # الحالة المقترحة
if self.use_importance:
    s_t = torch.sigmoid(self.importance_head(h_prev))  # درجة 0→1
    # h = s_t * حالة_قديمة + (1-s_t) * حالة_جديدة (بوابة ناعمة)
    h = s_t * self.A(h_prev) + (1 - s_t) * self.B(x_t)
    # احتفظ بـ s_t لاحتمال الهدف المساعد أو التحليل
    importance_scores.append(s_t)
else:
    h = torch.tanh(h_candidate)
```

> **ملاحظة التصميم:** استخدمنا `(1-s_t)` للمدخل الجديد — هذا يضمن
> أن المجموع موزون دائماً بين الماضي والحاضر. عندما s=1: نحتفظ بالماضي
> كاملاً. عندما s=0: نأخذ المدخل فقط.

**الخطوة ج — تحقّق من عدد المعاملات الإضافية:**
```python
def count_extra_params(model_base, model_imp):

```

**معيار القبول (بالأرقام):**
1. بوجود الوسم، الدرجات `s_t` كلها داخل [0,1] (اطبع min/max/mean).
2. زيادة المعاملات بسبب الوسم **< 1%** (اطبع الفرق بالأرقام).
3. معيار التدرّج عند الخطوة 10: بين 0.01 و 10 (اطبعه).
4. الرامة لا تزيد عن 300 م.ب. عن الإصدار بلا وسم.

**أخطاء شائعة ومعالجتها:**

| الخطأ | السبب | الحل |
|-------|-------|------|
| `nan` في s_t | مدخل كبير لـ sigmoid | أضف layer normalization قبل importance_head |
| s_t = 0.5 دائماً | لم يتعلّم | فعّل sparsity loss (انظر خطة البديل أ) |
| shape mismatch | importance_head يخرج (B,T,1) لا (B,T,D) | تأكّد من الضرب element-wise بـ broadcasting |

### المهمة 1.4 — حساب ومقارنة perplexity بالاختبار الإحصائي

**ما هو perplexity (PPL)؟**
مقياس جودة لنماذج اللغة = `exp(متوسط خسارة التنبؤ)` على بيانات تحقّق
(validation) لم يتدرّب عليها. **الأقل أفضل.**

**حساب التحليل الإحصائي الكامل:**

```python
import pandas as pd, numpy as np
from scipy import stats

df = pd.read_csv('results/experiments.csv')
stage1 = df[df['experiment'].str.startswith('stage1')]

base = stage1[stage1['use_importance']==False]['val_ppl'].values
imp  = stage1[stage1['use_importance']==True]['val_ppl'].values

print("="*50)
print("📊 نتائج المرحلة 1 — وسم الأهمية")
print("="*50)
print(f"بلا وسم: متوسط={base.mean():.2f} ± {base.std():.2f}")
print(f"بوسم:    متوسط={imp.mean():.2f}  ± {imp.std():.2f}")

improvement = (base.mean() - imp.mean()) / base.mean() * 100
print(f"\nنسبة التحسّن: {improvement:.2f}%")

# اختبار t لعيّنتين مستقلتين
t_stat, p_val = stats.ttest_ind(base, imp)
print(f"اختبار t: t={t_stat:.3f}, p={p_val:.4f}")

# تفسير النتيجة
if improvement >= 1.0 and p_val < 0.1:
    print("\n✅ قرار: الوسم يُحسِّن بشكل معنوي → انتقل للمرحلة 2")
elif improvement >= 1.0 and p_val >= 0.1:
    print("\n⚠️ التحسّن موجود لكن غير مؤكّد إحصائياً → زِد خطوات التدريب")
else:
    print("\n❌ الوسم لم يُثبت نفعه بهامش ≥ 1% → طبّق الخطط البديلة")
```

**تفسير قيمة p (للفهم):**
- p < 0.05: نتيجة مؤكّدة علمياً (نجاح).
- 0.05 ≤ p < 0.1: نتيجة محتملة (مقبولة مع ٣ بذور فقط).
- p ≥ 0.1: النتيجة غير حاسمة (فشل).

**معيار القبول (بالأرقام):** انظر نقطة الفحص أدناه.

## 🔧 الخطط البديلة (مشاكل المرحلة 1)

### الخطة البديلة أ — الوسم لا يتعلّم / لا فرق

**أعراض التشخيص:** s_t ثابتة ≈ 0.5 أو ≈ 0 طوال التدريب.

**الحل — فعّل الهدف المساعد (Sparsity Loss):**
```python
# أضف داخل حلقة التدريب:
LAMBDA_SPARSITY = 0.01  # يُمكن تجربة 0.001 → 0.1
if use_importance and len(importance_scores) > 0:
    s_all = torch.stack(importance_scores, dim=1)  # (B, T, 1)
    sparsity_loss = s_all.mean()  # يريد أن يكون منخفضاً
    total_loss = pred_loss + LAMBDA_SPARSITY * sparsity_loss
else:
    total_loss = pred_loss
```

**تجربة قيم λ:** 0.001 → 0.01 → 0.1 (ابدأ بالأصغر).
