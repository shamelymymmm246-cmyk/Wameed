## 🎯 هدف المرحلة
إضافة «ذاكرة هرمية» من أربع طبقات للنموذج الناجح من المرحلة 1، بحيث
يحتفظ النموذج بـ«خلاصة» النص القديم حتى بعد مرور كلمات كثيرة، ونُثبت
بالأرقام أنه **يتذكّر بداية النص الطويل أفضل** من النموذج بلا ذاكرة.

---

## 🧠 خلفية مبسّطة + التعريف الدقيق

**المشكلة:** نموذج SSM البسيط حالته `h` صغيرة، فمع طول النص «ينسى»
البداية. هذا ليس عيباً في التصميم — بل قيد رياضي: حالة ثابتة الحجم
تضطر لـ«نسيان» القديم لاستيعاب الجديد.

**الحل:** ذاكرة من أربع طبقات، كل طبقة **أبطأ تحديثاً وأكثر تجريداً**
من التي تحتها — مثل دماغ يحتفظ بالتفاصيل لحظياً وبالخلاصة طويلاً.

---

## 🗂️ الطبقات الأربع (التصميم الكامل)

### الطبقة 1 — قصيرة المدى (Short-Term, M1)
- **الحجم:** `(batch, state_dim)` — نفس حجم h الحالي.
- **تتحدّث متى؟** كل خطوة زمنية — مثل RAM الكمبيوتر.
- **ماذا تخزّن؟** حالة SSM الخام الأخيرة.
- **معادلة التحديث:**
  ```
  M1_t = h_t
  ```
  (مباشرة — هي نفس الحالة الآنية)

---

### الطبقة 2 — العاملة (Working Memory, M2)
- **الحجم:** `(batch, state_dim)` — نفس الحجم.
- **تتحدّث متى؟** كل `k=16` خطوة.
- **ماذا تخزّن؟** متوسط الحالات الـ 16 الأخيرة (Window Average).
- **معادلة التحديث:**
  ```python
  if step % k == 0:
      g2 = sigmoid(W_g2 @ M1 + b_g2)      # بوابة: كم نأخذ من الجديد؟
      summary = mean(M1_buffer[-k:])        # متوسط آخر k حالة
      M2 = (1 - g2) * M2 + g2 * summary   # دمج بوابي
  ```

---

### الطبقة 3 — الدلالية (Semantic Memory, M3)
- **الحجم:** `(batch, state_dim)` — يمكن أن يكون أصغر نصفاً.
- **تتحدّث متى؟** عندما يتجاوز متوسط `s_t` للنافذة عتبة معيّنة (مثلاً 0.6).
  هنا يأتي دور وسم الأهمية من المرحلة 1: الأجزاء المهمة وحدها تُحفظ طويلاً.
- **معادلة التحديث:**
  ```python
  avg_importance = mean(importance_buffer[-k:])
  if avg_importance > THRESHOLD_3:  # مثلاً 0.6
      g3 = sigmoid(W_g3 @ M2 + b_g3)
      M3 = (1 - g3) * M3 + g3 * linear3(M2)  # linear3: يُلخّص/يضغط M2
  ```

---

### الطبقة 4 — المجرّدة (Abstract Memory, M4)
- **الحجم:** `(batch, state_dim // 2)` — أصغر.
- **تتحدّث متى؟** نادراً — كل `K=256` خطوة، أو في نهاية كل مستند.
- **ماذا تخزّن؟** خلاصة الخلاصة — «سياق عام» للنص كله.
- **معادلة التحديث:**
  ```python
  if step % K == 0 or end_of_document:
      g4 = sigmoid(W_g4 @ M3 + b_g4)
      M4 = (1 - g4) * M4 + g4 * linear4(M3)
  ```

---

### قاعدة القراءة (كيف يستفيد النموذج)

عند كل خطوة، يدمج النموذج حالته الحالية h مع قراءة من الطبقات الأربع:

```python
def read_memory(h, M1, M2, M3, M4):
    """يُنتج تمثيلاً مدمجاً من الحالة والذاكرة"""
    # توحيد الأبعاد بطبقات خطية
    r1 = proj1(M1)   # (B, D)
    r2 = proj2(M2)   # (B, D)
    r3 = proj3(M3)   # (B, D)
    r4 = proj4(M4)   # (B, D)
    
    # دمج بالتسلسل ثم إسقاط
    combined = torch.cat([h, r1, r2, r3, r4], dim=-1)  # (B, 5D)
    return fusion(combined)   # (B, D) — طبقة خطية تُعيد D
```

## 🔢 المعاملات المُوصى بها

| المعامل | القيمة | السبب |
|---------|--------|-------|
| k (تحديث M2) | 16 | ~جملة متوسطة |
| K (تحديث M4) | 256 | نهاية فقرة |
| عتبة M3 (threshold_3) | 0.6 | فقط الأجزاء عالية الأهمية |
| حجم M3 | state_dim / 2 | ضغط طفيف |
| حجم M4 | state_dim / 4 | ضغط أكبر |
| تهيئة كل M | أصفار | نبدأ من ذاكرة فارغة |
| تهيئة W_g* | Xavier Normal | يمنع s≈0 أو s≈1 منذ البداية |

### المهمة 2.1 — بناء الطبقات الأربع مع قاعدة تحديث كل منها

**الخطوات التفصيلية:**

**أنشئ `wameed/engine/memory.py`:**

```python
"""
وحدة الذاكرة الهرمية لوميض
أربع طبقات: قصيرة → عاملة → دلالية → مجرّدة
"""
import torch
import torch.nn as nn

class HierarchicalMemory(nn.Module):
    def __init__(self, state_dim, k=16, K=256, threshold3=0.6):
        super().__init__()
        self.state_dim = state_dim
        self.k = k          # تردّد تحديث M2
        self.K = K          # تردّد تحديث M4
        self.thr3 = threshold3
        
        # بوابات الكتابة (write gates)
        self.gate2 = nn.Linear(state_dim, state_dim)
        self.gate3 = nn.Linear(state_dim, state_dim)
        self.gate4 = nn.Linear(state_dim, state_dim // 2)
        
        # طبقات التلخيص
        self.summary3 = nn.Linear(state_dim, state_dim)
        self.summary4 = nn.Linear(state_dim, state_dim // 2)
        
        # طبقات الإسقاط للقراءة
        self.proj1 = nn.Linear(state_dim, state_dim)
        self.proj2 = nn.Linear(state_dim, state_dim)
        self.proj3 = nn.Linear(state_dim, state_dim)
        self.proj4 = nn.Linear(state_dim // 2, state_dim)
        self.fusion = nn.Linear(5 * state_dim, state_dim)
        
        # عدّادات التتبّع
        self.step_count = 0
        self.m2_updates = 0
        self.m3_updates = 0
        self.m4_updates = 0
        
        # بفرات لحساب المتوسطات
        self._m1_buffer = []
        self._imp_buffer = []
    
    def init_memory(self, batch_size, device):
        """تهيئة الذاكرة بأصفار في بداية كل تسلسل"""
        d = self.state_dim
        self.M1 = torch.zeros(batch_size, d, device=device)
        self.M2 = torch.zeros(batch_size, d, device=device)
        self.M3 = torch.zeros(batch_size, d, device=device)
        self.M4 = torch.zeros(batch_size, d // 2, device=device)
        self.step_count = 0
        self._m1_buffer.clear()
        self._imp_buffer.clear()
    
    def update(self, h_t, importance_t=None):
        """
        يُحدّث الطبقات الأربع حسب الجداول
        h_t: (batch, state_dim) — الحالة الحالية
        importance_t: (batch, 1) — درجة الأهمية (من المرحلة 1)
        """
        self.step_count += 1
        
        # --- الطبقة 1: كل خطوة ---
        self.M1 = h_t.detach()  # .detach() لتجنّب تراكم التدرّجات
        self._m1_buffer.append(self.M1.clone())
        if len(self._m1_buffer) > self.k:
            self._m1_buffer.pop(0)
        
        # --- الطبقة 2: كل k خطوة ---
        if self.step_count % self.k == 0:
            self.m2_updates += 1
            summary = torch.stack(self._m1_buffer, dim=0).mean(dim=0)
            g2 = torch.sigmoid(self.gate2(self.M1))
            self.M2 = (1 - g2) * self.M2 + g2 * summary
        
        # --- الطبقة 3: عند أهمية عالية ---
        if importance_t is not None:
            self._imp_buffer.append(importance_t.squeeze(-1))
            if len(self._imp_buffer) > self.k:
                self._imp_buffer.pop(0)
            
            avg_imp = torch.stack(self._imp_buffer, dim=0).mean(dim=0).mean()
            if avg_imp.item() > self.thr3:
                self.m3_updates += 1
                g3 = torch.sigmoid(self.gate3(self.M2))
                self.M3 = (1 - g3) * self.M3 + g3 * torch.tanh(self.summary3(self.M2))
        
        # --- الطبقة 4: كل K خطوة ---
        if self.step_count % self.K == 0:
            self.m4_updates += 1
            g4 = torch.sigmoid(self.gate4(self.M3))
            self.M4 = (1 - g4) * self.M4 + g4 * torch.tanh(self.summary4(self.M3))
    
    def read(self, h_t):
        """
        يُنتج تمثيلاً مدمجاً من h والذاكرة الأربع
        يُعاد: (batch, state_dim)
        """
        r1 = torch.tanh(self.proj1(self.M1))
        r2 = torch.tanh(self.proj2(self.M2))
        r3 = torch.tanh(self.proj3(self.M3))
        r4 = torch.tanh(self.proj4(self.M4))
        combined = torch.cat([h_t, r1, r2, r3, r4], dim=-1)
        return torch.tanh(self.fusion(combined))
    
    def print_update_counts(self):
        """يطبع عدّادات التحديث للتحقّق"""
        print(f"  طبقة 1 (M1): {self.step_count} تحديث")
        print(f"  طبقة 2 (M2): {self.m2_updates} تحديث")
        print(f"  طبقة 3 (M3): {self.m3_updates} تحديث")
        print(f"  طبقة 4 (M4): {self.m4_updates} تحديث")
```

### المهمة 2.3 — اختبار التذكّر على النصوص الطويلة

**تصميم اختبار «المفتاح في البداية» (Key-Position Probe):**

```python
"""
اختبار التذكّر: نضع «مفتاحاً» في البداية ونقيس هل النموذج يتذكّره
بعد مرور توكنات كثيرة.
"""
def probe_test(model, tokenizer, probe_len=512, filler_len=400):
    """
    probe_len: الطول الكلي للنص
    filler_len: عدد التوكنات «الحشو» بين المفتاح والسؤال
    """
    # بناء النص: مفتاح + حشو + سؤال
    key_token = tokenizer.encode("MARKER")[0]  # رمز مميّز
    filler = tokenizer.encode(" the " * filler_len)[:filler_len]
    
    # نص بالمفتاح في البداية
    text_with_key = [key_token] + filler + [key_token]
    # نص بدون مفتاح (خط أساسي)
    text_no_key   = [filler[0]] + filler + [key_token]
    
    input_with_key = torch.tensor([text_with_key])
    input_no_key   = torch.tensor([text_no_key])
    
    model.eval()
    with torch.no_grad():
        loss_with_key = model(input_with_key).loss
        loss_no_key   = model(input_no_key).loss
    
    # إذا كان النموذج «يتذكّر» المفتاح، الخسارة ستكون أقل مع المفتاح
    improvement = (loss_no_key - loss_with_key) / loss_no_key * 100
    print(f"تحسّن PPL من وجود المفتاح: {improvement:.2f}%")
    return improvement
```

**أو: مقارنة PPL على الجزء الأول من نصوص طويلة:**

```python
def first_tokens_ppl(model, dataset, n_first=50, seq_len=512):
    """
    يحسب PPL على أول n_first توكن من نصوص بطول seq_len
    الفكرة: نموذج جيد الذاكرة يتنبأ بما في البداية أفضل
    بعد قراءة النص كاملاً
    """
    total_loss, count = 0.0, 0
    model.eval()
    for batch in dataset:
        # اقرأ النص كاملاً ثم قيّم على أول n_first فقط
        with torch.no_grad():
            out = model(batch['input_ids'])  # يعالج كل التسلسل
        # خسارة على التوكنات الأولى فقط
        loss = out.loss_per_token[:, :n_first].mean()
        total_loss += loss.item()
        count += 1
    return torch.exp(torch.tensor(total_loss / count)).item()
```

## 🔧 الخطط البديلة (مشاكل المرحلة 2)

### البديل أ — الذاكرة لا تساعد على النصوص القصيرة

**هذا طبيعي جداً.** الذاكرة الهرمية تظهر فائدتها فقط على النصوص الطويلة.
**الحل:** استخدم نصوصاً بطول 512-1024 توكن على Colab.

---

### البديل ب — M3 لا تُكتب أبداً

**التشخيص:** m3_updates = 0 رغم التدريب.

**الحل:** خفّض العتبة: `threshold3 = 0.6 → 0.4 → 0.2`

---

### البديل ج — البوابة g دائماً ≈ 0 (تنسى بسرعة)

**التشخيص:** M2 و M3 لا تتغيّران (طباعة نورم الفرق بين التحديثات).

**الحل:**
```python
# تهيئة بوابات الكتابة بتحيّز إيجابي (تسمح بالكتابة افتراضياً)
nn.init.constant_(self.gate2.bias, 1.0)  # sigmoid(1)≈0.73
nn.init.constant_(self.gate3.bias, 0.5)
```

---

### البديل د — الرامة تمتلئ

**الحل:**
- قلّل حجم M3 و M4 إلى `state_dim // 4`.
- قلّل طول التسلسل واذهب للـ Colab.

---

### البديل هـ — التدرّجات لا تصل للذاكرة

**التشخيص:** معيار التدرّج لـ memory parameters قريب من الصفر دائماً.

**الحل:** تأكّد من:
- `self.memory` جزء من `nn.Module` (مسجّل في parameters).
- لا `detach()` زائدة على مسار التدرّج.
- ريلو باستخدام `tanh` (لا relu) في دوال التلخيص لتحسين تدفّق التدرّج.
